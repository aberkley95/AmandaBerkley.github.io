SQL Data Cleaning
Project Explanation

SQL Script

After collecting the raw data I used MySQL to clean and standardize the data. I first created a second cities table to manipulate to ensure I still had a backup of the original data.
Throughout this process I not only found leading zeros and some random periods, but there were a couple duplicates and a null value as well.
First, I used the TRIM function for removing any leading and trailing zeros. I then used SELECT DISTINCT for various columns, including the “state” column, where I found multiple “California” inputs. This is where I noticed an unintentional period distinguishing it as a separate input. I used UPDATE and SET to fix this issue.
I also noticed some of the names of the columns were weird. For instance “ï»¿State” rather than “state”. I used ALTER TABLE and RENAME to fix any column names that seemed weird or bulky. Once all of the columns appeared standardized, I moved on to removing duplicates and null values.
I created a “city ID” column and used window functions to find duplicate data. From there I used INNER JOIN to delete these duplicates while leaving an original, which was made possible using the city ID column.
